# 测试文件目录

本目录包含项目开发过程中使用的测试和诊断工具文件。

## 📁 文件说明

这些文件是在开发和调试过程中创建的辅助工具，用于：
- 测试网络连接
- 诊断 URL 配置问题
- 验证环境配置
- 测试登录功能

## ⚠️ 注意事项

**这些文件不会影响小程序的正常运行**，它们已被移至此目录以保持项目结构清晰。

如果您不需要这些测试工具，可以安全地删除整个 `tests` 目录。

## 🗑️ 如何删除

如果确定不需要这些测试文件，可以直接删除整个 `tests` 目录：

```bash
# 在项目根目录执行
rm -rf mp-weixin/tests
```

## 📝 文件列表

- `test_*.js` - 各种测试脚本
- `diagnose-url.js` - URL 诊断工具
- `diagnostic_tool.js` - 综合诊断工具
- `request_fix.js` - 旧版请求修复器（已被 mini_fix.js 替代）
- `verify*.js` - 验证工具
- `find_default_account.js` - 查找默认账号工具

---

**提示**：现在项目使用统一的环境配置系统（`config.env.js`），这些测试工具已不再需要。
